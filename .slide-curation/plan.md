# 슬라이드 맥락 삽입 큐레이션 계획

## 🎯 목표
각 주차 md 노트의 **섹션 흐름을 방해하지 않으면서**, 해당 섹션의 개념을 시각적으로 보강하는 PDF 슬라이드만 선별 삽입.

## 📏 선별 기준 (교육적 가치)

**✅ 포함 (IN)**
- 핵심 다이어그램 (포인터 연결, 스택 동작, 원형 큐 인덱스, 네트워크 아키텍처 등)
- 개념 비교 표 (스택 vs 큐, CNN vs RNN 등)
- 수식 유도·증명 과정 (T(n), 빅오, Cross-Entropy, Backprop 등)
- 트레이싱 예시 (step-by-step 상태 변화)
- 교수님이 별표 치며 강조한 개념이 그림으로 그려진 페이지
- 알고리즘 의사코드가 도식과 함께 설명된 페이지

**❌ 제외 (OUT)**
- 표지, 목차, 로드맵, "Next Week"
- 단순 텍스트 bullet만 있는 페이지 (md에 이미 모든 텍스트 있음)
- 이전 주차 복습·반복
- 참고문헌, 크레딧
- 중복 슬라이드 (개념은 같고 미세 배치만 다름 → 대표 1장만)

**📊 예상 채택률** — 각 PDF에서 10~25% 페이지

## 🗂️ Phase 우선순위

| Phase | 주차 | 이유 |
|---|---|---|
| A-1 | DS 5차시 (연결 리스트) | 🌟 시험 8문제 예고 |
| A-2 | DS 6차시 (스택·수식) | 🌟 시험 3문제 예고 |
| A-3 | DS 7차시 (큐) | 🌟 족집게 3가지 |
| A-4 | DL CNN | 🌟 시스템 설계 문제 |
| A-5 | DL RNN | 🌟 교수님 오픈 질문 |
| B-1~3 | DS 2·3·4차시 | 핵심 개념 |
| B-4~5 | DL FNN/DNN·Overview | 기초 |
| C-1 | DS 1차시 | OT |
| C-2~8 | DL 잔여 + ML 전체 | 보완 |

## 🔁 진행 방식 (재개 가능)

1. `state.json` 읽어 다음 pending 주차 확인
2. 해당 PDF를 여러 배치로 Read (20p씩)
3. md 섹션 구조 확인
4. 판단 기준에 따라 매칭 결정
5. `js/slide-map.js`에 추가 (기존 내용 보존)
6. `state.json` 업데이트 (completed에 추가)
7. 커밋 + push
8. TodoWrite 업데이트
9. 다음 주차로

**중간 중단 시**: state.json이 진실의 원천. 재개 시 state에서 이어감.
