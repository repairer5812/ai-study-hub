## 문제 1 — 전통적인 기계학습과 딥러닝의 차이점

### 핵심 키워드 (채점 포인트)

- Feature Extraction 주체: `사람(전문가) → 핸드크래프트` vs `기계 → 자동 추출`
- 전통 ML 도구: `SIFT/HOG/Gabor` · `SVM/Random Forest/XGBoost` · `Kernel`
- 딥러닝 핵심: `End-to-End` · `Hidden Layer 계층적 학습` · `Edge → Motif → Object`
- 데이터·연산 환경: `대규모 데이터 + GPU 등장`
- 양면 비교: `딥러닝이 불리한 경우(소규모 데이터 / 해석 가능성 / 정형 데이터)`
- 일반화(Generalization)의 의미

### 모범답안

전통 기계학습과 딥러닝의 가장 본질적인 차이는 Feature Extraction(특징 추출) 을 누가 담당하느냐에 있다. 단순히 "딥러닝이 더 깊다" 같은 표면적 차이가 아니라, 학습 파이프라인 전체의 무게중심이 어디 놓이느냐가 다르다.

전통적인 기계학습 파이프라인은 보통 "원시 데이터 → 사람이 설계한 특징 추출 → 학습 알고리즘 → 출력"의 4단계로 이루어진다. 여기서 "사람이 설계한 특징 추출"이 가장 큰 노동이 들어가는 부분이었다. 컴퓨터 비전을 예로 들면, 28×28 MNIST 이미지를 그대로 Fully-Connected 모델에 넣을 수 있을지언정, 실용 시스템에서는 도메인 전문가가 SIFT, HOG, Gabor 필터, 7×7 컨볼루션 마스크 같은 핸드크래프트(hand-crafted) 특징을 직접 정의해 입력에서 의미 있는 부분을 뽑아냈다. 음성 인식에서는 MFCC, 자연어에서는 TF-IDF·n-gram, 시계열에서는 푸리에 변환·웨이블렛이 같은 역할을 했다. 그렇게 뽑힌 저차원 특징 벡터를 SVM, Random Forest, XGBoost, k-NN, 로지스틱 회귀 같은 학습 알고리즘에 넣어 분류·회귀를 수행했다. SVM의 경우 사용자가 RBF 같은 Kernel 함수를 미리 정의하고 서포트 벡터를 학습하는 식으로, "함수 형태는 사람이, 어떤 샘플이 중요한지는 데이터가" 결정하는 절충적 방식이었다. 이 패러다임에서 머신러닝 엔지니어의 가장 중요한 역량은 Feature Engineering, 즉 도메인 지식을 활용해 좋은 특징을 설계하는 능력이었다.

딥러닝은 이 구조를 근본적으로 바꿨다. 입력 → 여러 Hidden Layer → 출력이라는 단일 신경망 안에서 Feature Extraction과 Classifier를 동시에 학습한다. 이를 End-to-End 학습이라 부른다. 사람이 특징을 설계할 필요가 없다. 대신 Hidden Layer가 계층적으로 의미를 추출한다. 이미지 인식 망의 경우 첫 레이어는 Edge 같은 저수준 특징, 중간 레이어는 Motif·Texture, 깊은 레이어는 Object·Part 같은 고수준 의미를 자동으로 학습한다는 사실이 시각화 실험(Zeiler & Fergus, 2014)으로 확인되었다. 교수님 강의에서도 "얼굴 인식 예시에서 첫 레이어는 모서리, 그다음 레이어는 눈·코·입 같은 부분, 마지막은 얼굴 전체로 나아간다"고 설명되었으며, 이는 인간 시각피질의 V1 → V2 → V4 처리 단계와 흡사하다는 생물학적 타당성도 함께 가진다. 즉, 사람이 설계해야 했던 도메인 의존적 작업이 데이터로부터 자동 학습되는 표현 학습(Representation Learning) 으로 대체된 것이다.

이 차이를 가능하게 한 외부 조건은 두 가지다. 첫째, 대규모 데이터셋(ImageNet 1,400만 장 등)의 등장. 둘째, GPU 기반 병렬 연산의 발전. 딥러닝은 파라미터 수가 수백만~수십억 개로 매우 많아 충분한 데이터와 연산 자원이 없으면 학습이 불가능하다. 2010년대 이전에는 두 조건 모두 부재했고 그래서 깊은 망이 실용적이지 않았다. 2012년 AlexNet이 ImageNet에서 SVM 기반 시스템을 압도하면서 패러다임이 전환되었다.

수식적으로 보면, 전통 ML의 가설은 보통 다음과 같은 2단계 형태다.

$$y = g(\phi(x); W)$$

여기서 $\phi(\cdot)$은 사람이 미리 정의한 특징 함수, $W$는 학습 대상 파라미터다. 딥러닝의 가설은 같은 식이지만 $\phi$ 자체가 학습 대상이 된다.

$$y = g(\phi_W(x))$$

심지어 $\phi$를 여러 레이어로 분해하여 $\phi_W = \phi_W^{(L)} \circ \cdots \circ \phi_W^{(1)}$ 로 깊게 쌓을 수 있고, 모든 레이어가 한 번의 역전파(Backpropagation)로 동시에 학습된다. 이 단순한 구조 변경이 가져온 파급력이 크다.

다만 양면 비교가 중요하다. 딥러닝이 항상 우월한 것은 아니며, 다음과 같은 상황에서는 오히려 전통 ML (Machine Learning)이 더 적합하다.

첫째, 소규모 데이터셋이다. 딥러닝은 파라미터가 매우 많아 수천~수만 개 이상의 학습 데이터가 있어야 일반화된다. 의료 희귀 질환 분류처럼 데이터가 수십~수백 개에 불과한 문제에서는 핸드크래프트 특징 + SVM/Random Forest가 안정적으로 더 좋은 성능을 낸다.

둘째, 해석 가능성(Interpretability) 이 중요한 영역이다. 금융 신용 평가, 의료 진단처럼 "왜 이 결정을 내렸는지" 설명이 필수인 경우 딥러닝의 블랙박스 특성은 큰 부담이다. 결정 트리·로지스틱 회귀 같은 전통 ML은 각 feature의 기여도를 수식과 도표로 직접 보여줄 수 있어 규제·감사·신뢰성 측면에서 유리하다.

셋째, 정형 데이터(Tabular data) 다. 행과 열로 구성된 표 형태 데이터에는 깊은 망의 계층적 패턴 추출 강점이 잘 살지 않는다. Kaggle 정형 데이터 대회를 보면 여전히 XGBoost·LightGBM 같은 Gradient Boosting 모델이 우세하며, 딥러닝 기반 TabNet 같은 모델도 특정 조건에서만 경쟁력 있다.

마지막으로 일반화(Generalization) 라는 학습의 본질은 두 패러다임 모두 동일하다. 학습의 목표는 훈련 데이터를 외우는 것(Overfitting)이 아니라 새 데이터에서도 잘 작동하는 일반 규칙을 찾는 것이다. 전통 ML은 모델 복잡도를 사람이 통제(특징 선택, 모델 크기 조절)하고, 딥러닝은 Regularization, Dropout, Data Augmentation, Early Stopping 등으로 통제한다. 즉 도구는 다르지만 일반화 추구라는 목표는 같다.

요약하면, 두 패러다임의 본질적 차이는 (1) Feature Extraction의 주체가 사람에서 기계로 옮겨갔고, (2) 그 결과 도메인 지식 의존성이 줄고 End-to-End 학습이 가능해졌으며, (3) 대신 대규모 데이터·연산이 전제 조건이 되었다는 것이다. 그러나 작은 데이터·해석 가능성 요구·정형 데이터에서는 여전히 전통 ML이 강세이며, 두 패러다임은 경쟁이라기보다 상황에 맞게 선택·조합하는 도구라고 봐야 한다.

### 자주 빼먹는 함정

- "딥러닝이 더 깊다 / 레이어가 많다"만 적고 Feature Extraction 주체 차이라는 본질을 놓침
- 대규모 데이터 + GPU라는 외부 조건을 빠뜨림 (왜 2010년대에야 부흥했는지 설명 못 함)
- 양면 비교가 빠짐. 교수님은 "딥러닝이 불리한 경우"까지 답하는 것을 선호한다고 누차 강조
- 일반화의 의미 설명 없이 단순히 "정확도가 높다"로 끝냄

## 문제 2 — 다음 중 하나를 푸시오

> 두 옵션 모두 모범답안을 제시한다. 시험장에서는 둘 중 한 문제만 선택해 답하면 된다.

### 옵션 (1) — 오토인코더(Auto-Encoder)의 활용방안

#### 핵심 키워드

- 구조: `Encoder + Decoder` · `Bottleneck` · `Target = 입력 자기 자신`
- 학습 방식: `Self-supervised` · `라벨 불필요`
- Pre-training: `Greedy Layer-wise` · `Hinton 2006` · `Vanishing Gradient 우회`
- Denoising / Sparse / Variational AE · `생성 모델로의 발전`
- 차원 축소·이상치 탐지·표현 학습 활용

#### 모범답안

오토인코더(Auto-Encoder, AE)는 입력을 압축했다가 다시 복원하는 신경망으로, 출력층의 Target이 입력 자기 자신이라는 특이한 구조를 가진다. 라벨이 필요 없으므로 Self-supervised 학습 방식이며, Encoder와 Decoder 두 부분으로 구성된다.

구조는 다음과 같다. Encoder $f_\theta : x \to z$ 는 고차원 입력 $x \in \mathbb{R}^d$ 를 저차원 잠재 표현(Latent Representation) $z \in \mathbb{R}^k$ ($k < d$)로 압축한다. Decoder $g_\phi : z \to \hat{x}$ 는 그 표현으로부터 원본을 복원한다. 학습 목표는 복원 오차(Reconstruction Loss)를 최소화하는 것이다.

$$L_{\text{AE}}(\theta, \phi) = \mathbb{E}_{x \sim D} \big[ \| x - g_\phi(f_\theta(x)) \|^2 \big]$$

중간의 좁은 차원 $z$를 Bottleneck이라 부르며, 이 강제된 정보 병목 때문에 모델은 입력의 본질적 특징(잡음 제거된 압축 표현)을 스스로 학습하게 된다. 입력을 그냥 통과시킬 수 없으니 정말 중요한 정보만 골라 담는 것이다.

오토인코더의 활용방안은 크게 네 가지로 정리할 수 있다.

첫째, 가장 역사적 의의가 큰 활용은 깊은 신경망의 Greedy Layer-wise Pre-training이다. 2006년 이전에는 깊은 망을 쌓아도 Vanishing Gradient(Sigmoid·Tanh 미분값이 0.25 이하라 역전파 시 0으로 수렴) 때문에 입력층 근처 가중치가 학습되지 않아 깊은 망이 실용적으로 학습 불가능했다. Hinton(2006)은 "전체를 한 번에 학습시키지 말고 레이어별로 잘라서 사전학습하자"는 아이디어를 제안했다. 절차는 다음과 같다.

1. 원본 데이터로 첫 번째 오토인코더 학습 → Encoder 부분만 남기고 Decoder는 버림 → 첫 Hidden Layer 가중치 $W_1$ 고정.
2. 고정된 $W_1$의 출력을 입력으로 받아 두 번째 오토인코더 학습 → $W_2$ 확보.
3. 원하는 깊이까지 반복하며 $W_3, W_4, \ldots$ 쌓음.
4. 마지막에 Classification 레이어를 붙여 전체 망을 한 번에 지도학습으로 Fine-tuning.

이 방식은 각 레이어가 이미 의미 있는 초기값을 가진 상태에서 시작하므로 Vanishing Gradient의 영향이 훨씬 작다. 처음으로 깊은 신경망이 실제로 학습되어 딥러닝 부흥의 기폭제가 되었다. 이후 ReLU (Rectified Linear Unit, 2012)와 Batch Normalization(2015)이 Vanishing Gradient를 더 근본적으로 해결하면서 Pre-training 기법의 비중은 줄었지만, 딥러닝 재도약의 결정적 전환점이라는 역사적 의의는 여전히 크다.

둘째, 비선형 차원 축소(Non-linear Dimensionality Reduction) 다. PCA가 선형 변환만 다루는 데 비해 오토인코더는 비선형 함수(여러 Hidden Layer)를 사용해 더 복잡한 데이터 매니폴드를 잠재공간 $z$로 압축할 수 있다. 시각화·압축·검색용 임베딩으로 두루 활용된다. 실제로 Hinton(2006) Science 논문 "Reducing the Dimensionality of Data with Neural Networks"는 오토인코더가 PCA보다 훨씬 좋은 2D 임베딩을 만들 수 있음을 보였다.

셋째, 이상치 탐지(Anomaly Detection) 다. 정상 데이터로만 오토인코더를 학습시키면 정상 패턴을 잘 복원하도록 모델이 적응한다. 이후 새 데이터의 복원 오차 $\|x - \hat{x}\|^2$가 임계값보다 크면 "정상이 아닌 것"으로 판정한다. 산업 결함 검출, 신용 카드 사기 탐지, 네트워크 침입 탐지 등에서 라벨이 부족한 환경의 핵심 도구다.

넷째, 변형(Variant)과 생성 모델로의 발전이다.

- Denoising Auto-Encoder (DAE): 입력에 노이즈를 섞어 넣고 원본을 복원하도록 학습. 더 robust한 표현을 얻고 단순 복사를 방지한다.
- Sparse Auto-Encoder: 잠재 표현 $z$에 희소성(대부분 원소가 0) 제약을 주어 의미 있는 소수의 특징만 활성화되도록 한다.
- Variational Auto-Encoder (VAE): 잠재 표현을 확률 분포로 모델링하고 KL Divergence를 추가해 잘 정렬된 잠재공간을 학습한다. 이로써 단순 복원이 아니라 새 샘플 생성이 가능해진다. VAE는 GAN과 함께 생성 모델의 양대 산맥을 이루었다.
- Diffusion Model: 최근 Stable Diffusion 등 대유행한 생성 모델 계열도 노이즈를 점진적으로 제거하는 일종의 거대한 Denoising Auto-Encoder로 볼 수 있다. 즉 오토인코더의 사상은 현재 생성 AI의 기반에도 살아 있다.

요약하면, 오토인코더는 (1) 깊은 망 학습을 가능케 한 역사적 도구, (2) 비선형 차원 축소·표현 학습 도구, (3) 이상치 탐지 도구, (4) 현대 생성 모델(VAE·Diffusion)의 이론적 뿌리라는 네 축에서 활용된다. 라벨 없는 데이터로 의미 있는 표현을 학습할 수 있다는 점이 모든 활용의 공통 전제이며, 이는 Self-supervised Learning의 가장 초기 형태 중 하나로 평가된다.

#### 자주 빼먹는 함정

- 구조만 설명하고 "활용방안"을 풀지 않음. 문제는 Pre-training 등 활용을 묻고 있음
- Pre-training의 역사적 의의(Vanishing Gradient 우회, 딥러닝 부흥 기폭제)를 단순히 "초기값을 잘 준다"로 단축
- Variant(Denoising/VAE)와 현대 생성 모델로의 발전을 빼먹어 문제의 폭을 좁힘
- "Self-supervised, 라벨 불필요"라는 핵심 학습 방식 미언급

### 옵션 (2) — RNN의 파라미터 공유 메커니즘과 그것을 안 했을 때 일어나는 일

#### 핵심 키워드

- $h_t = f_W(h_{t-1}, x_t)$ · `same function, same parameters`
- Unfold: 시각화는 여러 블록, 실제 $W$는 단 하나
- 공유의 효과: `Arbitrary Length` · `파라미터 수 일정`
- 공유 안 할 때: `학습 시 길이 $T$를 넘는 시퀀스에 적용 불가`
- BPTT Gradient: $\partial L / \partial W = \sum_t \partial L_t / \partial W$ (공유 변수의 미분 원칙)
- 부산물: $W_{hh}$ 반복 곱 → Vanishing/Exploding Gradient 동기

#### 모범답안

RNN(Recurrent Neural Network)이 시퀀스 데이터를 처리할 수 있는 이유의 절반은 State Vector라는 메모리 개념이고, 나머지 절반은 모든 타임스텝에서 동일한 가중치 행렬 $W$를 공유한다는 파라미터 공유(Parameter Sharing) 원칙이다. 이 원칙이 무너지면 RNN의 핵심 장점이 통째로 사라진다.

(가) 파라미터 공유의 메커니즘

RNN의 상태 전이는 단일 함수로 정의된다.

$$h_t = f_W(h_{t-1}, x_t)$$

가장 간단한 Vanilla RNN의 경우 $f_W$가 다음과 같이 풀린다.

$$h_t = \tanh(W_{hh} \, h_{t-1} + W_{xh} \, x_t + b_h), \quad y_t = W_{hy} \, h_t + b_y$$

여기서 핵심 원칙은 강의 슬라이드의 표현 그대로 "The same function and the same set of parameters are used at every time step"이다. 즉 $t = 1, 2, 3, \ldots$ 어느 시점이든 동일한 함수 $f_W$를 반복 호출하며, 함수 내부의 가중치 $W_{xh}, W_{hh}, W_{hy}$는 모든 시점이 같은 메모리 주소를 공유한다.

시간축으로 펼친 Unfold 그림을 보면 마치 여러 개의 블록이 줄지어 있는 듯 보인다. 그러나 그것은 시각화일 뿐, 실제로는 단 하나의 $W$를 $T$번 사용하는 것이다. 마치 같은 도장을 시간을 따라 여러 번 찍는 것과 같다. State Vector $h_t$가 "처음부터 $t-1$까지의 모든 히스토리가 압축된 인코딩 메모리(Encoded Memory)" 역할을 하면서 동일 함수가 시간을 따라 흐를 수 있게 한다.

이 공유에는 두 가지 즉각적 효과가 있다.

첫째, 파라미터 수가 시퀀스 길이와 무관하게 일정하다. 100-step이든 1,000-step이든 학습할 가중치 수는 동일하다. 그래서 매우 긴 시퀀스도 훈련 가능하다.

둘째, 그리고 더 중요하게, Arbitrary Length(임의 길이) 시퀀스 처리가 가능하다. 학습 시에 본 적 없는 새 길이의 시퀀스도 같은 $W$를 재사용해 그대로 처리할 수 있다. 이는 음성 인식·기계 번역·텍스트 생성처럼 매번 입력 길이가 다른 모든 시퀀스 태스크에서 결정적인 능력이며, RNN이 CNN (Convolutional Neural Network)·MLP (Multi-Layer Perceptron) 대비 가지는 핵심 차별점이다.

(나) 파라미터를 공유하지 않으면 일어나는 일

만약 시점마다 별도의 가중치 $W_1, W_2, \ldots, W_T$를 두면 두 가지 결정적 문제가 발생한다.

첫째, 파라미터 수의 폭증과 학습 비효율이다. 시퀀스 길이가 $T$일 때 파라미터 수가 $T$배로 늘어난다. 같은 학습 데이터로 $T$배 많은 가중치를 학습해야 하므로 데이터 효율이 매우 떨어지고 과적합 위험이 폭증한다. 또한 시점마다 다른 $W$를 학습하므로, 시점 5에서 배운 패턴이 시점 6에서는 전혀 활용되지 않는다는 비효율이 생긴다.

둘째, 그리고 본질적으로 더 치명적인 문제는 학습 길이 $T$를 넘어서는 시퀀스에 모델 자체가 적용 불가능해진다는 점이다. 학습 시 시퀀스 길이를 $T = 50$으로 잡았다면 $W_1 \sim W_{50}$만 학습된다. 그런데 테스트 단계에서 $T' = 100$짜리 시퀀스가 들어오면 $W_{51} \sim W_{100}$이 존재하지 않아 모델이 그냥 작동하지 않는다.

이는 RNN의 가장 중요한 본질, 즉 Arbitrary Length 처리 능력을 통째로 무너뜨린다. 음성 인식 시스템을 예로 들면, 학습 시에는 5초 발화로 학습했는데 실사용 시 10초짜리 발화가 들어오면 기능이 멈춰버린다. 이는 실용적으로 절대 받아들일 수 없는 문제다. 결과적으로 파라미터 공유를 포기하면 RNN은 사실상 길이가 고정된 일반 MLP와 다를 바 없는 모델이 된다 — 그것도 매우 비효율적인 형태로.

반대로 $W$를 공유하면 시퀀스를 100번 반복하든 1,000번 반복하든 동일 $W$를 그대로 재사용하므로 길이 제약이 자연스럽게 사라진다. 시간축의 일관성(시점이 달라도 같은 변환 규칙이 작동한다는 가정)이 모델 구조에 반영된 결과다.

(다) BPTT에서 Gradient가 합산되는 이유

총 손실은 보통 각 시점 손실의 합이다.

$$L = \sum_{t=1}^{T} L_t$$

$W$가 모든 시점에서 공유되므로 Chain Rule에 의해 $W$에 대한 gradient는 각 시점 기여분의 합이 된다.

$$\frac{\partial L}{\partial W} = \sum_{t=1}^{T} \frac{\partial L_t}{\partial W}$$

직관적으로 "하나의 $W$가 시점 1에서도, 시점 2에서도, …, 시점 $T$에서도 쓰였으니 각 시점에서 $W$가 기여한 gradient를 모두 더해주어야 올바른 업데이트 방향이 나온다." 이는 Computational Graph에서 "공유된 변수의 gradient는 그 변수가 등장한 모든 지점의 편미분을 합산한다"는 일반 원칙과 정확히 일치한다.

각 항 $\partial L_t / \partial W$를 더 깊이 풀면 시점 $t$에서 발생한 손실이 $h_t \to h_{t-1} \to \cdots \to h_1$으로 거슬러 올라가는 경로마다 $W_{hh}$가 반복적으로 곱해진다.

$$\frac{\partial h_t}{\partial h_k} = \prod_{i=k+1}^{t} \frac{\partial h_i}{\partial h_{i-1}}$$

이 곱셈에서 $W_{hh}$의 가장 큰 특이값이 1보다 크면 Exploding Gradient(발산), 1보다 작으면 Vanishing Gradient(0으로 소실)가 발생한다. 즉 파라미터 공유가 RNN의 본질이지만, 같은 $W$가 반복 곱해지는 구조 자체가 학습 불안정성의 원천이기도 하다는 점이 흥미로운 양면성이다. 이 구조적 약점이 LSTM (Long Short-Term Memory, 1997)·GRU (Gated Recurrent Unit, 2014) 같은 게이트 기반 RNN의 등장 동기가 되었으며, 더 최근에는 Transformer가 순환 구조 자체를 폐기하고 Self-Attention으로 대체하는 방향으로 진화했다.

(라) 정리

RNN의 파라미터 공유는 단순한 구현 디테일이 아니라 (1) Arbitrary Length 시퀀스 처리, (2) 데이터 효율, (3) 시간축 일관성 모델링이라는 세 핵심 능력의 기반이다. 이 공유를 포기하면 학습 길이를 넘는 시퀀스에 적용 불가능해지며, 이는 RNN을 시퀀스 모델로 정의하는 근본 자질이 무너지는 것과 같다. 한편 같은 $W$의 반복 곱셈은 Vanishing/Exploding Gradient 문제를 낳아 LSTM·GRU·Transformer로의 진화를 이끌었다.

#### 자주 빼먹는 함정

- "파라미터를 공유한다"만 쓰고 왜 공유해야 하는지(Arbitrary Length 처리)를 놓침
- 공유 안 할 때의 답을 "파라미터 수가 늘어난다"로 단축. 핵심은 학습 길이를 넘는 시퀀스에 적용 불가능하다는 점
- Gradient 합산이 "공유 변수의 미분 원칙"이라는 일반 결과임을 빠뜨리고 그냥 합한다고만 적음
- 양면성(같은 $W$ 반복 곱이 Vanishing의 원인이기도 함)을 언급해 LSTM 동기와 연결하면 가산점

## 문제 3 — 가변 크기 이미지에 대한 CNN+RNN 설계 및 장단점

> 입력 이미지 크기가 가변적이고, 모델 구조를 CNN+RNN으로 만든다.

### 핵심 키워드 (채점 포인트)

- 모델: `1:N (One-to-Many)` · `Image Captioning 패턴`
- CNN 인코더: `사전학습 VGG/ResNet` · `Global Average Pooling` (가변 크기 처리)
- VGG 마지막 Softmax·FC 제거 이유: `Classification → Description` 기능 차이
- 결합 방식: `이미지 임베딩 → RNN의 초기 State Vector $h_0$로 주입`
- RNN 디코더: `<Start>/<End> 토큰` · `Auto-regressive` · `Word Embedding`
- 장점: 가변 입출력 대응 / End-to-End 학습 / 모달 결합
- 단점: 오류 전파(Error Propagation) / GPU 병렬화 한계 / 학습 데이터 요구량

### 모범답안

문제는 입력 이미지 크기가 가변적이라는 제약과 출력으로 자연어 문장(가변 길이 시퀀스)을 생성한다는 가정 아래 CNN (Convolutional Neural Network)+RNN (Recurrent Neural Network) 결합 모델을 설계하라는 것이다. 이는 전형적인 Image Captioning 태스크이며, RNN의 입출력 구조 분류로 보면 1:N (One-to-Many) — 정적인 입력 1개에서 시퀀스 출력을 만드는 — 형태에 해당한다.

(가) 전체 구조 설계

설계는 크게 세 모듈로 구성된다.

1. CNN 인코더 (이미지 → 고정 차원 임베딩 벡터)

ImageNet 등 대규모 데이터셋으로 사전학습된(Pre-trained) CNN을 인코더로 사용한다. VGG (Visual Geometry Group network)-16/19, ResNet (Residual Network)-50/152, EfficientNet 같은 표준 모델 중 하나면 된다. 강의에서는 VGG를 예로 들었으므로 그 흐름을 따른다.

VGG의 원래 구조는 (Conv 블록들) → Flatten → FC (Fully-Connected) 4096 → FC 4096 → FC 1000 → Softmax 형태로 마지막에 1,000개 ImageNet 클래스 분류기가 붙어 있다. Image Captioning에서는 이 마지막 Softmax와 그 직전 FC 두 개(Classification head)를 제거한다. 그 직전, 즉 마지막 Convolutional Feature Map이나 그 풀링 결과를 임베딩으로 사용한다.

가변 크기 이미지를 처리하기 위한 핵심 장치는 Global Average Pooling(GAP) 이다. 일반적으로 VGG는 224×224 입력을 가정해 마지막 FC가 고정 크기 입력을 받지만, GAP를 쓰면 마지막 Conv Feature Map의 공간 차원(예: 7×7 또는 14×14)이 어떻게 변하든 채널별 평균을 취해 항상 동일한 차원(예: 512차원) 임베딩 벡터 $v$를 얻을 수 있다.

$$v_c = \frac{1}{H' \cdot W'} \sum_{i,j} F_{c,i,j}, \quad c = 1, \ldots, C$$

여기서 $F$는 마지막 Conv Feature Map, $H', W'$는 임의의 공간 차원이다. 결과 $v \in \mathbb{R}^C$는 입력 이미지 크기에 무관하다.

2. 결합 점 (이미지 임베딩 → RNN의 초기 State Vector)

추출된 고정 차원 임베딩 $v$를 RNN 디코더의 초기 State Vector $h_0$로 주입한다.

$$h_0 = \tanh(W_{ih} \, v + b_{ih})$$

또는 $v$를 직접 $h_0$에 매핑하거나, 매 타임스텝의 입력에 concat해 넣는 방식도 가능하다. 핵심 발상은 "이 이미지가 담고 있는 의미"가 RNN의 시작점, 즉 초기 메모리 컨디션이 된다는 것이다. 이후 RNN은 이 임베딩을 기반으로 단어를 하나씩 생성한다.

3. RNN 디코더 (임베딩 → 가변 길이 문장)

RNN 디코더는 LSTM (Long Short-Term Memory) 또는 GRU (Gated Recurrent Unit)를 사용하는 것이 일반적이다. 7주차에서 본 Vanilla RNN은 Vanishing Gradient 때문에 긴 문장 생성이 어렵다. 디코더의 입력은 매 타임스텝마다 단어 임베딩(Word Embedding)이며, 첫 타임스텝의 입력은 `<Start>` 특수 토큰이다. 그 시점의 hidden state는 위에서 주입한 $h_0$다.

매 타임스텝 출력은 어휘(Vocabulary) 전체에 대한 Softmax 확률 분포이며, 가장 확률 높은 단어(Argmax)나 분포대로 샘플링한 단어를 생성한다.

$$y_t = \text{Softmax}(W_{hy} \, h_t + b_y)$$

생성된 단어를 다음 시점의 입력 $x_{t+1}$로 되먹이는 Auto-regressive 방식으로 문장을 한 단어씩 만들어 나간다.

$$h_t = f_W(h_{t-1}, x_t), \quad x_{t+1} = \text{Embed}(y_t)$$

이 과정은 모델이 `<End>` 토큰을 생성하는 시점에 종료된다. 이로써 출력 문장의 길이도 가변이 된다. 학습 시에는 정답 문장의 단어들을 입력으로 그대로 사용하는 Teacher Forcing 기법을 쓰며, 강건성을 위해 모델이 직접 샘플링한 단어를 입력으로 섞어 넣기도 한다(Scheduled Sampling).

전체 손실 함수는 모든 시점의 Cross-Entropy를 합한 형태다.

$$L = -\sum_{t=1}^{T} \log p(y_t^* | y_{<t}, v)$$

여기서 $y_t^*$는 시점 $t$의 정답 단어다.

(나) VGG의 마지막 Softmax·FC를 제거하는 이유

이 부분은 시험에서 가장 자주 묻는 핵심 디테일이다. VGG의 최종 Softmax와 그 직전 FC Hidden 레이어는 1,000개 ImageNet 클래스로 분류(Classification) 하는 데만 특화되어 있다. 이 단계에서는 "개·고양이·버스" 같은 카테고리 라벨에 필요한 핵심 정보만 남기고 나머지는 압축·제거된다. 그러나 이미지를 풍부하게 묘사(Description) 하는 데 필요한 색감, 구도, 객체 간 관계, 배경 같은 다양한 하이레벨 특징은 이 단계에서 이미 손실된다.

캡셔닝은 분류가 아니라 묘사다. 같은 "개" 클래스라도 "갈색 강아지가 잔디 위에서 공을 물고 있다"는 다양한 의미가 필요하다. 따라서 Classification head를 버리고, 그 직전의 더 일반적이고 풍부한 Convolutional Feature Map(또는 그 GAP 결과)을 임베딩으로 사용해야 한다. 이 결정이 잘못되면 캡션이 천편일률이 된다 — "개", "차", "사람"식의 단순 분류 라벨처럼 출력되는 것이다.

(다) 장단점 분석

장점

1. 가변 입출력 대응이 자연스럽다. CNN의 Global Average Pooling으로 입력 이미지 크기 제약을 없앴고, RNN의 Auto-regressive 생성과 `<End>` 토큰으로 출력 문장 길이도 가변으로 처리된다. 두 모달 모두에서 가변성을 다룰 수 있는 흔치 않은 구조다.

2. 모달 결합과 End-to-End 학습이 가능하다. CNN의 공간 인식력(Local Pattern, Translation Invariance)과 RNN의 시퀀스 생성력(Memory, Auto-regressive)을 각자의 강점에 맞게 분담시킨다. 이미지 인코더에는 사전학습된 가중치를 활용하므로 적은 캡션 데이터로도 빠르게 수렴한다. 또한 손실이 RNN 출력에서 CNN까지 한 번에 역전파되어 두 모듈이 같이 최적화된다.

3. 사전학습 모델 활용으로 학습 효율이 좋다. ImageNet으로 학습된 VGG는 일반화 성능이 매우 뛰어나 다양한 이미지에서 즉시 작동하며, 캡션 데이터(MS COCO 등)는 이미지 라벨 데이터보다 훨씬 적어도 충분히 학습이 된다.

단점

1. 오류 전파(Error Propagation) 가 심하다. RNN 디코더는 Auto-regressive 방식이라 $t-1$ 시점의 잘못된 단어를 $t$ 시점의 입력으로 받는다. 초반 단어 하나가 어긋나면 뒷문장 전체가 틀린 방향으로 흘러간다. 학습 시 Teacher Forcing으로 정답을 입력해도 추론 시에는 자기 출력을 입력으로 받아야 해서 Train-Test Mismatch(Exposure Bias) 문제도 발생한다.

2. GPU 병렬화 제한이다. RNN의 본질적 순차 의존성($h_t$ 계산에 $h_{t-1}$ 필수)으로 시퀀스 생성 속도가 길이에 비례한다. 길이 100인 문장이라면 100번 순차 연산해야 하므로 GPU의 병렬성을 활용할 수 없다. 대량 추론 환경(예: 실시간 서비스)에서 병목이 된다. Transformer 기반 캡셔너(예: Show, Attend and Tell의 후속 BLIP·CLIP 결합 모델)가 등장한 직접 동기다.

3. 단일 글로벌 임베딩의 한계다. 이미지 전체를 GAP로 평균해 한 벡터로 만들면 객체 간 공간 관계, 작은 디테일 등이 평균되어 흐려진다. "오른쪽 아래에 있는 개"처럼 위치 정보가 필요한 묘사가 어려워진다. 후속 연구는 이를 해결하기 위해 Attention Mechanism을 도입했다 — 매 단어 생성 시점에 이미지 어느 부분을 봐야 할지 동적으로 가중치를 매기는 방식(Show, Attend and Tell, 2015)이며, 이는 Transformer의 Self-Attention으로 이어지는 진화 경로다.

4. 학습 데이터 요구량이 크다. 이미지-캡션 쌍 데이터(MS COCO, Flickr30k 등)는 라벨 데이터보다 구축 비용이 훨씬 높다. 또한 캡션은 한 이미지에 여러 정답이 가능해 평가 지표(BLEU, METEOR, CIDEr 등)도 지도학습 분류만큼 깔끔하지 않다.

(라) 정리

전체 구조는 (1) 가변 크기 이미지를 사전학습 VGG + Global Average Pooling으로 고정 차원 임베딩 벡터 $v$로 압축하고, (2) $v$를 LSTM/GRU 디코더의 초기 State Vector로 주입한 뒤, (3) `<Start>` 토큰부터 시작해 Auto-regressive 방식으로 단어를 생성, `<End>` 토큰에서 종료하는 1:N RNN 모델이다. 마지막 Softmax·FC를 제거하는 이유는 그 부분이 분류 특화라 묘사용 풍부한 특징이 소실되기 때문이다. 장점은 가변 입출력 대응과 사전학습 활용 효율이며, 단점은 오류 전파, GPU 병렬화 한계, 글로벌 임베딩의 디테일 손실이다. 이 한계들이 후속 Attention 기법과 Transformer 기반 모델로의 진화 동기가 되었다.

### 자주 빼먹는 함정

- "VGG 쓴다"만 적고 왜 마지막 Softmax·FC를 버리는지 이유(Classification → Description 전환)를 생략. 교수님이 가장 자주 강조한 디테일
- 가변 이미지 크기 처리에 Global Average Pooling이 핵심 장치라는 점 미언급
- `<Start>`/`<End>` 토큰을 빠뜨리고 "RNN이 알아서 시작·종료한다"고 적음
- Auto-regressive 구조에서 자기 출력을 입력으로 되먹이는 흐름을 명시 안 함
- 단점에서 "느리다"만 적고 GPU 병렬화 한계라는 구조적 이유 미설명
- Attention 기반 후속 모델(Show, Attend and Tell)로의 발전 가능성을 언급하면 답안의 폭이 넓어져 가산점
