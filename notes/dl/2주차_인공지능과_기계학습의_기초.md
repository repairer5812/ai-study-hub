# 🚀 [딥러닝 2주차] 인공지능과 기계학습의 기초 (AI & ML In a Nutshell)

## 1. 🤖 인공지능(AI)의 정의와 에이전틱 AI (Agentic AI)

### 1.1. AI의 기본 정의 (CS 관점)
- 환경(Environment)을 인지(Perceive)하고, 목표 달성 가능성을 최대화하는 행동(Action)을 취하는 유연하고 합리적인 에이전트(Agent).

### 💡 핵심 출제 포인트: 기존 딥러닝/머신러닝과 Agentic AI의 차이점
- 기존 딥러닝/머신러닝: 주로 데이터의 패턴을 찾고 예측/분류하는 것에 집중 (액션의 개념이 없음).
- Agentic AI: 인지(Perception) $\rightarrow$ 추론(Reasoning) $\rightarrow$ 행동(Action) 이 들어간다는 것이 가장 큰 차이점입니다. (교수님 강조 및 메모 내용)

---

## 2. 💻 전통적 프로그래밍 vs 기계학습 (Traditional Programming vs. ML)

### 2.1. 전통적 프로그래밍 (Traditional Programming)
- 입력: 데이터(Data) + 프로그램(Program, 룰/알고리즘)
- 출력: 결과물(Output)
- 특징: 프로그래머가 직접 if, for 문 등을 디자인해야 함.

### 2.2. 기계학습 (Machine Learning)
- 입력: 데이터(Data) + 원하는 출력/정답(Desired Output)
- 출력: 알고리즘 (Algorithm/Program) 💡 (메모 강조 포인트)
- 특징: 데이터에 숨겨진 패턴을 찾아내는 알고리즘을 기계가 스스로 만들어냄. 데이터가 바뀌어도 패턴을 찾는 메커니즘 자체는 바뀌지 않음.

## 3. 🤔 기계학습은 언제/왜 사용하는가? (Why Machine Learning?)

### 3.1. 예시: MINST 손글씨 숫자 인식 데이터셋
- 28x28 픽셀 = 총 784차원 데이터.
- 문제점: 숫자 0~9를 쓰는 사람마다 모양이 조금씩 다르고 완벽하게 똑같은 숫자가 존재하지 않음 (Variation이 큼).
- 결론: 핸드크래프팅(수작업 룰)으로 모든 예외 상황을 정의할 수 없기 때문에 기계학습을 사용함. 반대로, 데이터에 베리에이션이 없다면 굳이 ML을 쓸 필요가 없음 (오히려 손해).

### 3.2. 기계학습이 적합한 4가지 케이스
1. 사람이 룰을 설명할 수 없는 경우: 음성 인식, 행동 인식 (우리가 듣고 이해하지만 그 '룰'을 컴퓨터에게 설명할 수는 없음).
2. 전문가 지식 자체가 존재하지 않는 경우: 화성 탐사 등 사전 지식이 없는 분야.
3. 솔루션이 시간에 따라 변하는 경우: 컴퓨터 네트워크 라우팅.
4. 특정 케이스에 맞춰 적응해야 하는 경우: 바이오메트릭스 (얼굴 인식 - 나이가 들거나 안경을 쓰는 등 시간에 따라 변화함).

## 4. 🧠 기계학습에서의 '학습(Learning)'과 파이프라인

### 4.1. 학습의 정의
- 데이터로부터 자동으로 정보를 추출하고 분석할 수 있는 컴퓨터 모델을 구축하는 것.

### 💡 핵심 출제 포인트: 일반화 (Generalization)
- 단순히 주어진 트레이닝 데이터를 완벽하게 외우는 것(Overfitting)이 목표가 아님.
- 데이터에 공통적으로 들어있는 일반화된 룰을 찾아, 처음 보는 새로운 데이터(Test data)에서도 정확하게 예측할 수 있는 능력이 가장 중요함.

### 4.2. Feature Extraction (특징 추출)
- 전통적 컴퓨터 비전/ML: 784차원 이미지를 그대로 쓰면 노이즈(배경 등)가 많아, 사람(전문가)이 직접 필터(예: 7x7 마스크)를 씌워 유의미한 특징을 뽑아냄. 기계학습에서 가장 많은 노력이 들어갔던 부분.
- Deep Learning과의 차이: 딥러닝 시대가 오면서 이 Feature Extraction 과정을 기계가 알아서 하게 되어 사람이 직접 할 필요가 없어짐. 💡 (메모 강조 포인트)

---

## 5. 🗂️ 기계학습의 종류 및 모델

### 5.1. Supervised Learning (지도 학습): 타겟(정답) 값이 존재.
- Regression (회귀): 연속적인(Continuous) 값 출력.
- Classification (분류): 이산적인(Discrete/Category) 값 출력.

### 5.2. Unsupervised Learning (비지도 학습): 타겟 값이 없음.
- Clustering (군집화), Visualization (시각화).
- Density Estimation (분포 추정): 가우시안 믹스처 모델(GMM) 등을 사용하여 전체 데이터의 분포를 추정. (데이터 생성 모델로 활용 가능). 💡

### 5.3. 기타 주요 모델 (메모 기반 출제 포인트)
- Hidden Markov Model (HMM): 시계열 데이터(Time-series) 처리에 사용.
- Support Vector Machine (SVM): 커널(Kernel)이라는 특수한 폼을 사용하여 선형(Linear) 및 비선형(Non-linear) 데이터 모두에 적용 가능. 💡

## 6. 📈 다항식 곡선 피팅 (Polynomial Curve Fitting) 예제

### 6.1. 데이터 생성 배경 (교수님 예시 & Q&A 포함)
- 입력(X) 1차원, 출력(Y) 1차원인 가장 쉬운 예제.
- 데이터 10개는 어떻게 만들었을까? 입력 $x$는 0과 1 사이의 균등 분포(Uniform Distribution)에서 랜덤하게 뽑음.
- 원래 정답 함수는 사인 함수($\sin(2\pi x)$)임.
- 하지만 현실 세계의 데이터는 완벽하지 않으므로, 사인 함수 값에 가우시안 랜덤 노이즈(Gaussian Random Noise)를 더해서 최종 10개의 관측 데이터(초록색 점)를 만듦.
- (교수님 TMI/Q&A): 가우시안 랜덤 넘버는 어떻게 만들까? 0~1 사이 균등 분포에서 난수를 뽑은 뒤, 가우시안의 누적분포함수(CDF)의 역함수에 집어넣으면 가우시안 샘플이 됨. 교수님은 대학원 시절 다변량(Multivariate) 랜덤 넘버 생성 코드를 직접 짜기도 하셨음.

### 6.2. 가설 설정 (Hypothesis)
- 우리는 정답이 사인 함수라는 것을 모름. 데이터를 보니 선형(직선)은 절대 아님.
- 가장 쉽게 생각할 수 있는 비선형 모델인 다항식(Polynomial)으로 가설을 세움.
    $$
    y(x, w) = w_0 + w_1x + w_2x^2 + \dots + w_Mx^M
    $$

### 💡 핵심 출제 포인트: W에 대한 선형성 (메모 반영)
- 이 식은 입력 $x$의 관점에서는 제곱, 세제곱이 들어가므로 분명한 비선형(Non-linear)임.
- 하지만 우리가 찾아야 하는 파라미터 $W$의 관점에서는 모두 1차식인 선형(Linear) 모델임.
- 왜 이게 중요할까? $W$에 대해 선형이기 때문에, 미분했을 때 유일한 글로벌 옵티멈(Global Optimal, 최적해)을 찾기 매우 쉽기 때문임.

## 7. 📉 손실 함수 (Loss Function / Error Function)

### 7.1. 개념
- 예측값과 실제 타겟값의 차이를 계산하여, 어떤 $W$가 더 좋은지 판단하는 기준.

### 💡 핵심 출제 포인트: 용어의 차이 (메모 반영)
- 전통적인 기계학습(ML) 관점: Error Function (에러 함수)
- 딥러닝(Deep Learning) 관점: Loss Function (손실 함수)

### 7.2. MSE (Mean Squared Error) 사용 이유
- 에러에는 양수와 음수가 섞여 있으므로 크기를 따지기 위해 절대값이나 제곱을 취함.
- 절대값은 미분 불가능한 뾰족한 지점이 생기기 때문에, 수학적으로 미분이 편한 제곱(Square)을 취함.

### 💡 핵심 출제 포인트: 글로벌 미니멈의 존재 (메모 반영)
- 선형 모델($W$ 관점)의 에러 함수는 제곱을 취하기 때문에 2차식 형태(볼록/오목한 포물선 형태)가 됨.
- 따라서 미분해서 0이 되는 지점이 딱 하나 존재하며, 이 Minimum 포인트(Global solution)가 유일하게 존재함.

## 8. 🧩 모델의 복잡도와 오버피팅 (Model Complexity & Overfitting)
- 차수 $M$을 몇으로 할 것인가? ($M=0, 1, 3, 9$)

### 💡 핵심 출제 포인트: M과 표현력의 관계 (메모 반영)
- $M$이 커지면(차수가 올라가면) 그래프가 구부러질 수 있는 기회가 $M-1$번 생김.
- 즉, 모델의 표현력(Capacity)이 좋아진다고 표현함.
- 하지만 $M=9$처럼 데이터 개수(10개)에 가깝게 차수를 높이면, 주어진 트레이닝 데이터를 완벽하게 지나가게 됨 (Error = 0).
- 이로 인해 새로운 데이터에 대한 예측력이 완전히 망가지는 오버피팅(Overfitting) 문제가 발생함.
- 오버피팅의 증상: 곡선이 모든 점을 지나가기 위해 심하게 요동치면서, 앞에 곱해지는 계수 $W$ 값들이 비정상적으로 매우 커짐.

### 💡 핵심 출제 포인트: 모델 설계의 기본 룰 (메모 반영)
- 처음부터 복잡한 모델(Function)을 테스트하지 않음.
- 가장 간단한 함수부터 테스트를 시작해서, 점진적으로 더 복잡한 함수로 테스트해보는 것이 기본 룰임.

---

## 9. 🛡️ 오버피팅을 해결하는 방법 (Regularization)

### 9.1. 방법 1: 데이터 늘리기
- 💡 핵심 출제 포인트 (메모 반영): 파라미터 숫자보다 데이터 숫자를 늘리면 오류가 줄어듦. (경험적으로 파라미터 수의 5~10배 이상의 데이터가 필요함).

### 9.2. 방법 2: 정규화 (Regularization)
- 오버피팅이 되면 $W$ 값이 비정상적으로 커지는 현상을 막기 위한 수학적 테크닉.
- 기존 에러 함수(Goodness of fit)에 페널티 항(Penalty term)을 더함.
    $$
    \tilde{E}(w) = \text{기존 에러} + \frac{\lambda}{2} ||w||^2
    $$
- 즉, 에러도 줄이면서 동시에 $W$의 크기도 커지지 않도록 억제하는 것.

### 💡 핵심 출제 포인트: 람다($\lambda$)와 Weight Decay (메모 반영)
- 람다($\lambda$): $W$ 값의 크기를 얼마나 중요하게 볼 것인지(페널티를 얼마나 강하게 줄 것인지)를 결정하는 가중치.
- 데이터에서 학습되는 값이 아니라 사용자가 직접 설정하는 하이퍼파라미터(Hyper-parameter)임.
- 딥러닝(Deep Learning)에서는 이 정규화 기법을 Weight Decay라고 부름.
- $\lambda$가 0이면 오버피팅, 너무 크면 $W$가 0에 수렴하여 언더피팅됨. 적절한 값을 찾아야 하며, 이를 위해 진짜 테스트 데이터가 아닌 검증용 데이터(Validation Data)를 사용해야 함.

## 10. 🧱 기저 함수 (Linear Basis Function Models)

### 10.1. 개념의 전환
- 입력 데이터 $X$를 그대로 쓰지 말고, 중간에 비선형 함수(Non-linear function)를 끼워 넣어 새로운 특징 공간으로 변환하자는 아이디어입니다. 이 중간 함수를 기저 함수(Basis Function, $\phi(x)$)라고 부릅니다.
- 수식의 변화:
    $$
    y(x, w) = w_0 + w_1\phi_1(x) + w_2\phi_2(x) + \dots = W^T\phi(x)
    $$

### 💡 핵심 출제 포인트: 여전히 선형 모델인 이유
- 입력 $X$를 기저 함수에 통과시켰기 때문에 $X$의 관점에서는 매우 복잡한 비선형(Non-linear) 함수가 됩니다.
- 하지만 우리가 찾아야 하는 파라미터 $W$의 관점에서는 여전히 선형(Linear) 결합입니다. 따라서 최적화(미분해서 해를 찾는 것)가 매우 쉽다는 엄청난 장점을 유지합니다.

### 10.2. 기저 함수의 종류 (교수님 예시 상세)
- 다항식 (Polynomial): $\phi_j(x) = x^j$
- 가우시안 / RBF (Radial Basis Function): 특정 평균(위치)에서 방사형(Radial)으로 퍼져나가는 종 모양의 함수. 거리에 따라 동일한 값을 가집니다. (확률 분포로 해석할 필요 없음)
- 시그모이드 (Sigmoid) / Tanh: S자 형태의 함수. Tanh는 시그모이드를 위아래(-1 ~ 1)로 늘어뜨린 형태입니다.
- 푸리에(Fourier) / 웨이블릿(Wavelet): 컴퓨터 비전에서 이미지 필터링(예: Gabor 필터)을 할 때 사람의 뇌에서 동작하는 방식과 유사하게 쓰이는 기저 함수입니다.
- 결론: 이러한 기저 함수들을 여러 개 섞어서 사용하면(Linear Combination), 별다른 노력 없이도 굉장히 복잡하고 다양한 함수 $F$를 디자인할 수 있습니다.

## 11. 📐 정규화(Regularization)의 기하학적 의미 (L1 vs L2)

앞선 Part 2에서 배운 정규화 수식 $\sum |w_j|^q$ 에서 $q$의 값에 따라 기하학적 모양과 성질이 완전히 달라집니다.

### 11.1. $q=2$ (Quadratic, L2 Norm, Ridge)
- 모양: 원(Circle) 형태.
- 특징: W 공간에서 원형으로 에러가 커집니다. 가장 일반적으로 많이 쓰이는 정규화 방식입니다.

### 11.2. $q=1$ (Absolute value, L1 Norm, Lasso)
- 모양: 마름모 / 다이아몬드 형태.

### 💡 핵심 출제 포인트: Lasso의 Feature Selection 역할
- 마름모 모양은 축 위에 뾰족한 모서리(Corner)가 존재합니다.
- 원래 에러 함수의 등고선과 이 마름모가 만나는 최적점이 축 위에서 형성될 확률이 매우 높습니다.
- 축 위에서 만난다는 것은 특정 $W$ 값이 정확히 0이 된다는 뜻입니다. (예: $W_1 = 0$)
- 즉, 불필요한 특징(Feature)에 곱해지는 가중치를 0으로 만들어버림으로써, 중요한 Feature가 무엇인지 기계가 스스로 선택(Selection)하게 해주는 엄청난 역할을 합니다.

### 11.3. Elastic Net (L1 + L2 혼합)
- Lasso(L1)는 유용하지만, 학습할 때마다 선택되는 Feature가 달라지는(불안정한) 단점이 있습니다.
- 이를 보완하기 위해 L1과 L2를 동시에 더해서 사용하는 방식을 기계학습에서는 Elastic Net이라고 부릅니다.

## 12. 🔄 회귀(Regression)에서 분류(Classification)로의 확장

### 💡 핵심 출제 포인트 (메모 완벽 반영)
- "Regression에 한 번 더 함수를 통해서 결과가 나오면 Classification이다."
- 회귀와 분류는 근본적으로 다르지 않습니다. 회귀 모델에서 나온 실수형 연속값(예: -10.5, 300.2 등)을 그대로 쓰지 않고, 0과 1 사이의 값으로 가둬주는(Bounding) 함수를 한 번 더 씌우면 그것이 분류 모델이 됩니다.

### 12.1. 활성화 함수 (Activation Function)
- 이때 씌워주는 함수를 활성화 함수라고 부릅니다. (예: 비선형 함수인 Sigmoid)
- 출력값이 0~1 사이로 나오면, 이를 '확률(Probability)'로 해석할 수 있습니다.

### 12.2. 손실 함수(Loss Function)의 변화: MSE $\rightarrow$ Cross-Entropy
- 출력값이 확률로 바뀌었기 때문에, 기존에 쓰던 MSE(에러 제곱합)는 더 이상 의미가 없습니다.
- 대신 우도(Likelihood) 개념을 도입합니다. 관측된 데이터가 해당 클래스일 확률을 모두 곱한 값입니다.
- 곱셈은 계산이 복잡하므로 로그(Log)를 씌워 덧셈으로 바꿉니다. 여기에 마이너스를 붙인 것이 바로 Cross-Entropy Error입니다.
- (교수님 TMI): Cross-Entropy라는 용어는 정보 이론(Information Theory)에서 왔습니다. 원래 엔트로피 정의가 $-p \log p$ 인데, 예측 확률과 실제 정답 확률 두 개가 교차(Cross)되어 들어가기 때문에 붙여진 이름입니다.

### 12.3. 다중 분류 (Multi-class Classification)
- 클래스가 3개 이상일 때는 Sigmoid 대신 Softmax 함수를 사용합니다.
- Softmax는 전체 출력값의 합이 무조건 1이 되도록(확률의 성질) 강제해 주는 역할을 합니다.

## 13. ⚙️ 기계학습 / 딥러닝 파이프라인 3단계 요약 (교수님 강조)
실제 딥러닝 코딩을 할 때 정확히 다음 3단계를 거치게 됩니다.

1. 모델/아키텍처 정의 (가설 설정): 어떤 함수 $F$ (어떤 레이어, 어떤 활성화 함수)를 쓸 것인가?
2. Loss Function 정의: MSE를 쓸 것인가, Cross-Entropy를 쓸 것인가? 여기에 어떤 정규화(Regularization, Weight Decay) 텀을 추가할 것인가?
3. Optimization (최적화) 호출: Gradient Descent 등 어떤 알고리즘으로 $W$를 업데이트할 것인가?

## 14. 🎯 최적화 기법 (Gradient-based Optimization)
- 경사 하강법 (Steepest Gradient Descent): 에러 함수의 기울기(Gradient)를 구한 뒤, 기울기의 반대 방향(가장 가파르게 내려가는 방향)으로 조금씩 이동하며 최적의 $W$를 찾는 방법.
- 학습률 (Learning Rate, $\eta$): 한 번에 얼마나 이동할지 결정하는 보폭. 너무 작으면 오래 걸리고, 너무 크면 진동하거나 발산함.
- 데이터 업데이트 방식:
    - Batch: 전체 데이터를 한 번에 보고 업데이트 (안정적이나 매우 느림).
    - Stochastic (SGD): 데이터 1개만 보고 업데이트 (빠르고 로컬 미니멈에서 빠져나올 가능성이 있으나 매우 노이즈가 심함).
    - Mini-batch SGD: 데이터를 적당한 묶음(예: 32개, 64개)으로 나누어 업데이트. (현재 딥러닝에서 가장 많이 쓰이는 훌륭한 타협안).
